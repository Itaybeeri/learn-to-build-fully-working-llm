# Contributing

Thanks for wanting to make this course better! Clearer explanations, small fixes, better analogies, and
new interactive demos are all very welcome.

## The one thing to know first: your progress is private

When you learn through the course, the tutor saves your progress in a **`.progress/`** folder. That folder
is listed in [`.gitignore`](.gitignore), so:

- It **never** appears in `git status`.
- `git add .` and `git add -A` **skip it**.
- It can only end up in a commit if you *deliberately* force it with `git add -f` — so **please don't**.

This means you can be a learner *and* a contributor from the same clone, and your personal progress will
never leak into a pull request. The same applies to trained model weights (`*.pt`, `*.npz`), which are also
gitignored — they're regenerated by running the demos.

## What lives where

- `docs/learn/**` — the course content (the notes, the glossary, the interactive portal, the Module 7
  demos). **This is what contributions change.**
- `CLAUDE.md` + `.claude/commands/` — the AI-tutor rulebook and the `/start`, `/continue`, `/recap`
  commands.
- `.progress/` — **not** course content; it's each learner's private state (gitignored).

## How to contribute

1. Fork the repo and create a branch.
2. Make your change to the course content.
3. Check your diff — confirm `.progress/` and any `*.pt` / `*.npz` files are **not** included.
4. Open a pull request describing what you improved and why.

## Style notes (so the manual stays consistent)

The notes are written to be a **standalone, in-order book** for a beginner:

- **Intuition first, then the key math**, with every symbol explained from zero.
- **No forward references** — never rely on a concept introduced later.
- **Keep the teaching voice** — plain language, and keep the guiding questions (the "Check yourself" and
  "Questions we worked through" sections) rather than stripping them out.
- Each concept is its own `.md` file; if you add one, add it to `docs/learn/index.md` too.

If you're adding a portal demo, follow the pattern in `docs/learn/portal/` (a `modules/moduleN.js` that
self-registers via `Portal.register({...})`, plus one `<script>` line in `index.html`).
